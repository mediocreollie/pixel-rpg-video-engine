# Asset Pipeline

This project will use placeholders until the video engine is stable. Final art, music, and sound should be added later through a deliberate production pass.

The goal is a Nintendo DS-era pixel RPG feel: stylised, readable on phones, warm, and expressive without becoming realistic.

## Characters

Real photos of people can be used as inspiration for character design, but final in-game characters should be stylised DS-era pixel characters.

Character art should aim to keep people recognisable through:

- Hair shape
- Outfit colours
- Posture
- Signature accessories
- Broad expression or vibe

Character art should avoid:

- Realistic facial detail
- Photo tracing
- Overly accurate likenesses
- High-detail animation

Planned character asset stages:

1. Reference gathering
2. Simple character design notes
3. Still portrait placeholder
4. Walking sprite sheet
5. Optional emote portraits

Walking sprites should be created later as small directional sheets suitable for top-down movement.

If real-person reference photos are used, store them safely outside the public repo when needed. Do not commit private or sensitive reference photos into `public/assets/`.

## Locations

Real photos can be used as visual inspiration for locations, but final locations should be reusable pixel RPG spaces.

Locations should be generic enough to support multiple scenes. A pub should work for many pub jokes, a street should work for many travel scenes, and a cafe should work for more than one coffee sketch.

Planned reusable locations:

- Pub
- Beach
- Bedroom
- Street
- Gym
- Cafe
- Town
- House
- Supermarket
- Park

Location art should focus on readable silhouettes, clear exits, and visual punchline space.

## Music

Use placeholder silence or temporary development-only loops for now.

Future music should use short loopable tracks with moods such as:

- Town
- Pub
- Beach
- Awkward moments
- Reveals
- Menu

Do not commit copyrighted music unless it is licensed for this project.

## Sound Design

Future sound effects should be small, readable, and useful for recording.

Planned sound effects:

- Dialogue blips
- Footsteps
- Door enter sound
- Scene transition sound
- Punchline reveal sting
- Menu selection sounds

Keep sounds short and non-intrusive so they support social video edits.

## UI Polish

Future UI polish should reinforce the DS-era RPG identity.

Planned UI polish:

- DS-era menu sounds
- Title card animation
- Dialogue box texture
- Small screen shake options
- Fade in/out transitions

These should stay optional and data-driven where practical.

## Naming

Use lowercase kebab-case filenames.

Examples:

- `friend-jack-walk.png`
- `beth-portrait.png`
- `town-loop.mp3`
- `door-enter.wav`
- `pub-ambience.mp3`
- `dialogue-blip-soft.wav`

Avoid spaces in asset filenames.
