# Pixel RPG Video Engine Art Direction

## Purpose

This document exists to guide artistic decisions.

The project already has a technical architecture.

The goal now is to create a world that feels memorable, believable, and worth exploring.

When artistic decisions conflict with technical convenience, prefer the option that improves atmosphere, readability, and storytelling.

---

# One Sentence Vision

Create a world that feels like:

**Pokemon Platinum exploration + Stardew Valley environmental richness + real-world Adelaide-inspired locations.**

This is not a Pokemon clone.

This is not a Stardew clone.

This is a fictionalised version of real life presented through a Pokemon-inspired RPG.

---

# The World Is The Main Character

The biggest mistake the project can make is focusing on characters instead of locations.

Players should remember:

* the pub
* the beach
* the gym
* the cafe
* the supermarket

not individual sprites.

The world delivers the joke.

The player discovers it.

---

# Visual Hierarchy

Priority order:

1. Environment
2. Location landmarks
3. NPCs
4. Player
5. UI

The player should never dominate the screen.

The environment should always feel larger than the player.

---

# Character Philosophy

## Scale

Characters should be small.

The player should feel like they exist inside the world.

Characters should generally occupy approximately 5-10% of the visible play space.

If a player sprite feels large, reduce it.

---

## Detail Level

Characters should be recognisable.

Not realistic.

Use:

* hair colour
* clothing colour
* silhouette
* accessories

Avoid:

* facial detail
* realistic proportions
* excessive animation

---

## Animation

Animation should feel Nintendo DS inspired.

Prioritise:

* readability
* responsiveness
* charm

Over:

* realism
* complexity

A strong two-frame walk cycle is better than a weak six-frame walk cycle.

---

# Environment Philosophy

## Density

Every location should feel lived in.

Avoid:

* empty floors
* empty walls
* large unused spaces
* giant colour blocks

Every room should contain:

* props
* furniture
* decoration
* points of interest

If an area feels empty, add environmental storytelling.

---

## Clutter

Good clutter:

* books
* signs
* stools
* plants
* shelves
* posters
* decorations
* glasses
* food
* gym equipment

Bad clutter:

* random noise
* meaningless decoration
* repetitive filler

Every object should suggest how people use the space.

---

# Interior Design

Interiors should feel cosy.

Primary references:

* warm taverns
* cafes
* pubs
* community spaces

Use:

* wood
* warm lighting
* furniture variation
* decoration

Avoid:

* giant empty rooms
* perfect symmetry
* sterile spaces

A room should tell a story before dialogue begins.

---

# Exterior Design

Exterior locations should feel inspired by real life.

Not fantasy kingdoms.

Not JRPG capitals.

Inspired by:

* Adelaide suburbs
* local beaches
* shopping strips
* parks
* cafes
* pubs
* gyms

Locations should be simplified, not copied.

---

# Colour Philosophy

Overall mood:

Warm
Inviting
Nostalgic

Avoid:

* harsh saturation
* neon colours
* pure black shadows

Use:

* warm browns
* soft greens
* sandy yellows
* muted blues

The world should feel comfortable to spend time in.

---

# Lighting

Lighting should create atmosphere.

Examples:

Pub

* warm orange light
* dark wood
* amber highlights

Beach

* bright sunlight
* warm sand
* soft blue water

Town

* daylight
* greenery
* subtle contrast

Night

* cool blue shadows
* warm windows

---

# Environmental Storytelling

Locations should communicate information without dialogue.

Example:

Pub

Bad:

* table
* beer

Good:

* tables
* stools
* taps
* glasses
* signs
* shelves
* groups of drinks

Players should instantly understand where they are.

---

# Comedy Philosophy

Comedy comes from escalation.

Not dialogue.

Not punchlines.

Not memes.

Structure:

1. Normal request
2. Journey
3. Reveal
4. Exaggeration

Example:

"Let's go to the pub."

Player follows.

Pub appears.

Entire pub contains absurd quantities of beer.

The environment delivers the joke.

---

# Real-Life Inspiration

Use real places.

Transform them into game locations.

Examples:

Glenelg -> Beach Area

Local Pub -> Social Hub

Rundle Mall -> Shopping Area

Local Gym -> Fitness Area

The goal is familiarity.

Not realism.

---

# Asset Creation Priorities

When deciding what to improve next:

Priority 1:
World readability

Priority 2:
Location identity

Priority 3:
Character readability

Priority 4:
Animation

Priority 5:
Visual polish

Never prioritise polish over readability.

---

# Rules For Future Development

Do not add visual features because they are technically impressive.

Add visual features because they improve:

* atmosphere
* storytelling
* exploration
* humour
* location identity

If a new asset does not make the world feel more alive, it is probably not important.

---

# Success Test

A screenshot should be understandable without context.

Someone looking at the screen should immediately know:

* where they are
* what kind of place it is
* where they can go
* what the joke is

without needing dialogue to explain it.

If the screenshot cannot achieve this, the environment needs more work.
